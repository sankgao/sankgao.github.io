像其它版本控制系统（VCS）一样，Git 可以给仓库历史中的某一个提交打上标签，以示重要。比较有代表性的是人们会使用这个功能来标记发布结点（`v1.0.0`、`v2.0.0` 等等）。

在发布一个版本时，通常先在版本库中打一个标签（`tag`），这样，就确定了打标签时刻的版本。将来无论什么时候，取某个标签的版本，就是把那个打标签的时刻的历史版本取出来。所以，标签也是版本库的一个快照。

Git 的标签其实就是指向某个 `commit` 的指针（跟分支很像，但分支可以移动，标签不能移动），所以，创建和删除标签都是瞬间完成的。
